### 🚩 里程碑 1：核心对话流（打通前后端）

**目标：** 实现像微信/ChatGPT 一样的基础聊天界面。

- **对话窗口：** 自动滚动到底部的聊天列表，区分“用户气泡”和“AI 气泡”。
- **输入框逻辑：** * 处理 `Enter` 发送、`Shift+Enter` 换行。
  - 发送时按钮变为 `Loading` 状态，防止重复点击。
- **流式渲染 (Streaming)：** **（进阶重点）** 不要等 AI 全部写完才显示，要实现一个字一个字蹦出来的效果。你需要处理 `ReadableStream`。
- **Markdown 支持：** 使用 `react-markdown` 渲染 AI 返回的内容（比如代码块高亮、加粗、表格）。

### 🚩 里程碑 2：用户系统与会话管理（接入数据库）

**目标：** 让你的应用有“记忆”，不再是刷新网页就没了。

- **身份认证 (Auth)：** * 登录/注册页面（表单校验）。
  - 把后端返回的 **JWT Token** 存在 `localStorage` 或 `Cookie` 中。
  - **Axios 拦截器：** 每次发请求自动带上 Token。
- **历史侧边栏：**
  - 左侧显示“历史会话列表”（从数据库拉取）。
  - 支持“新建会话”、“删除会话”、“修改会话标题”。
- **数据同步：** 点击左侧不同的会话 ID，中间的聊天窗口要实时切换并加载对应的历史消息。

### 🚩 里程碑 3：工程优化与部署（面试加分项）

**目标：** 证明你具备“生产环境”的考虑，回击 HR 问的独立部署能力。

- **错误处理：** 当后端崩了、API 没余额或网络断了时，给用户友好的弹窗提示（Toast）。
- **响应式适配：** 适配手机端（移动端聊天体验很重要）。
- **环境区分：** * 开发环境请求 `localhost:3000`。
  - 生产环境请求你的 **AWS 域名**。
- **部署实战：**
  - 使用 `Vercel` 部署 React 前端。
  - 配置 **Nginx 反向代理**，解决跨域（CORS）问题。


方案 A：极致 AI 路线（推荐：Python FastAPI + MySQL + Redis）

适合： 目标是 AI 后端、全栈开发、数据驱动型公司。
